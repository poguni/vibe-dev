2. 대상 : 초등학교 4, 5, 6학년 학생 및 교사

3. 핵심 기능
  ① AI가 결과를 정확하게 분석할 수 있도록 초등학생 수준에 맞는 평가 문항(30문항)을 만들어 주세요.

  ② 평가 문항을 제작할 때에는 아래 내용을 반영해 주세요.
      - 성격, 흥미, 가치관 등의 핵심 요소가 반영되어야 함
      - 학습 및 생활 습관을 설문 문항에 포함하여 분석 결과에 따른 조언을 할 수 있도록 생성
      - MBTI, 홀랜드 검사, 가치관 지표, 커리어넷(한국직업능력연구원)의 '초등학생용 진로발달검사 / 직업흥미검사' 등 기존에 있는 검사들을 활용

  ③ 설문에 대한 응답 내용을 바탕으로 AI가 결과를 분석해서 '진로검사 리포트'를 작성해야 합니다.
      - 실제 '분석 리포트'처럼 성격, 흥미, 가치관, 학습태도, 요약 등 분석 결과를 구체적으로 작성할 것 
      - 분석한 결과를 토대로 '추천 직업', '추천 학습 방법', '교사-학부모에게 하는 조언'도 함께 생성할 것 

  ④ 진로 검사 리포트는 'PDF로 다운로드 / 리포트 인쇄' 가 가능해야 합니다.

  ⑤ 설문을 시작하기 전 간단한 로그인 페이지를 생성해 주세요. '학교명', '학년', '반', '이름'을 입력하게 하고, 그 밑에는 '진로 여행 출발하기!' 버튼을 만들어 주세요.

  ⑥ 교사용 관리자 페이지 : 브라우저 로컬 저장소(LocalStorage) 및 관리자 페이지 연동
      - 현재 개발 중인 앱 내에 '선생님용 관리 대시보드'를 추가
      - 학생이 검사를 완료하면 결과 데이터(학교, 학년, 반, 이름, 답변, 검사 결과)가 브라우저의 LocalStorage에 자동으로 누적 저장됩니다.
      - 로그인 화면 구석에 '교사용 관리자 로그인' 버튼을 만들고, 클릭하면 선생님 전용 관리 페이지로 진입합니다.
      - 관리자 로그인 아이디는 'admin', 비밀번호는 'admin1204'로 해 주세요.
      - 관리 페이지에서는 종합 결과 요약 테이블과 우리반 학생들의 검사 결과 목록을 한눈에 볼 수 있어야 합니다.
      - 클릭 한 번으로 각 학생의 리포트를 다시 열어보거나 인쇄(PDF 저장)할 수 있으며, 전체 데이터를 **엑셀(xlsx) 파일로 내보내기(Export)**할 수 있어야 합니다.
      - 엑셀로 저장할 때에는 세로 열의 너비를 데이터의 길이에 알맞게 충분히 크게 늘려 주세요.

[데이터 저장 방식 변경]

Local Storage → (Google Apps Script) → Google Sheet

  현재 발생하고 있는 문제는 **로컬 스토리지(localStorage)**의 특성 때문에 발생합니다. 로컬 스토리지는 데이터를 입력한 기기의 웹 브라우저 내에만 격리되어 저장되므로, 다른 컴퓨터(교사 PC)에서 접근할 수 없습니다.
  이를 해결하기 위해 **Google Sheets(구글 스프레드시트)**를 데이터베이스로 삼고, **Google Apps Script(GAS)**를 중간 API 서버로 구축하는 방식의 솔루션을 제안합니다.

  1. 데이터 저장 (학생 → Sheets): 학생이 검사를 완료하면, 브라우저가 GAS 웹 앱 URL로 POST 요청을 보내 결과를 Google Sheet의 새 행으로 추가합니다.
  2. 데이터 조회 (Sheets → 교사 대시보드): 교사가 관리자 로그인 후 대시보드에 접근하면, app.js가 GAS 웹 앱 URL로 GET 요청을 보내 모든 학생의 결과 데이터를 실시간으로 가져와 화면을 그립니다.
